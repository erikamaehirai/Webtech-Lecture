Request Header Field

Request Header Field Name Description Example
Accept Content type that are accepable for the response Accept: text/plain
Accept-Language Human languages for response Accept-Language: en-US
Accept-Charset Character sets that are acceptable. Accept-Charset: utf-8
Authorization Authentication credentials for HTTP Authorization: BASIC Z3Vlc3Q6Z3Vlc3QxMjM=
Proxy-Authorization Authorization credentials for connecting to proxy Proxy-Authorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==
Expect Indicates that particular server behaviors are required by the client. Expect: 100-continue
From email address of the user making the request. From: user@example.com
Host Domain name of the server Host: www.slu.edu.ph
If-Match If the client supplied an entity matches the same entity on the server If-Match: "xyzzy", "r2d2xxxx", "c3piozzzz"
If-None-Match if one of the given value in this tag matches the given entity tags represented by ETag. If-None-Match: "xyzzy"
If-Modified-Since Only send the response if the entity has been modified since a specific time If-Modified-Since: Sun, 19 Mar 2017 19:43:31 GMT
If-Unmodified-Since Only send the response if the entity has not been modified since a specific time. If-Unmodified-Since: Sun, 19 Mar 2017 19:43:31 GMT
If-Range Partial Content If-Range: Sun, 19 Mar 2017 19:43:31 GMT
Max-Forwards Control where the trace has to be clone Max-Forwards: 10
Transfer-Encoding Indicates what extension transfer-coding it is willing to accept in the response TE: deflate
User-Agent Identify the request User-Agent: Mozilla/4.0 (compatible; MSIE5.01; Windows NT)
Referer Address of the previous web page from which a link to the currently requested page was followed. Referer: http://www.webtek.org/http/index.htm
